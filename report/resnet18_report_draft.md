# ResNet-18 Robustness Report for CIFAKE

## Abstract
This report summarizes ResNet-18 performance on CIFAKE using previously generated outputs. The model achieved high clean performance, with 95.99% accuracy and 95.94% F1 score. However, robustness analysis under JPEG compression, Gaussian blur, and Gaussian noise shows non-uniform degradation. In particular, Gaussian blur and high Gaussian noise caused the largest drops, indicating that high clean accuracy does not guarantee robustness.

## Introduction
Detecting synthetic images requires not only strong clean-data performance but also stability under realistic image corruptions. This draft presents a reproducible visualization workflow that reads existing ResNet-18 outputs and generates figures for reporting.

## Dataset
CIFAKE is a binary image classification dataset with 32x32 RGB images:
- REAL images from CIFAR-10 (label 0)
- FAKE images generated by Stable Diffusion (label 1)

## Methodology
A ResNet-18 architecture (adapted for 32x32 inputs) was previously trained and evaluated. This workflow does not retrain the model. It only reads existing artifacts:
- clean metrics CSV
- robustness metrics CSV
- training log CSV
- confusion matrix PNG

Corruptions use fixed settings:
- JPEG: 75, 50, 25
- Gaussian blur: sigma 1, 2, 3
- Gaussian noise: sigma 0.05, 0.10, 0.20

## ResNet-18 Training Summary
Training/validation curves are plotted from `training_log.csv` to summarize convergence behavior across epochs.

## Clean Evaluation
Known clean evaluation metrics:
- Accuracy: 95.99%
- F1 score: 95.94%

These values indicate strong separation on uncorrupted test images.

## Robustness Analysis
Robustness curves and average-drop summaries highlight sensitivity to increasing corruption severity. The model shows limited degradation under mild distortions but larger degradation in stronger perturbations, especially:
- Gaussian blur at higher sigma
- Gaussian noise at sigma 0.20

Average drop is computed as:

`drop = clean_accuracy - corrupted_accuracy`

## Discussion
The results support two practical observations. First, clean accuracy alone is insufficient for deployment reliability. Second, corruption-specific sensitivity matters: blur and strong noise can significantly reduce classifier confidence and decision quality, even when clean performance is strong.

## Conclusion
ResNet-18 provides strong clean CIFAKE performance, but robustness remains a key limitation. Future work should include robustness-aware training and augmentations, calibration checks, and comparison with alternative architectures.
